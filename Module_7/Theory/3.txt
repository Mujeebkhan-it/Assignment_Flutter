Q - 3. Explain the purpose of HTTP methods (GET, POST, PUT, DELETE) and when to use each.

HTTP Methods (GET, POST, PUT, DELETE)

1. GET (Retrieve Data):

Purpose: Fetch data from the server without modifying it.
Use: Use for reading or retrieving resources (e.g., get user details).
Example: GET /users/{id}


2. POST (Create Data):

Purpose: Send data to the server to create a new resource.
Use: Use for creating new items (e.g., adding a new user).
Example: POST /users



3. PUT (Update Data):

Purpose: Update an existing resource with new data.
Use: Use for replacing or updating a resource (e.g., update user details).
Example: PUT /users/{id}


4.DELETE (Remove Data):

Purpose: Remove a resource from the server.
Use: Use for deleting resources (e.g., deleting a user).
Example: DELETE /users/{id}
Each method serves a specific purpose for managing resources in a RESTful API.
